=== PARA 20 [STYLE:List Paragraph] ===

=== PARA 21 [STYLE:Heading 2] ===
1.2 Our Solution — LuminFlow

=== PARA 22 [STYLE:Normal] ===
LuminFlow is an intelligent audit transparency platform that addresses all six Challenge 2 priorities through an integrated, AI-powered visualization framework:

=== PARA 23 [STYLE:List Paragraph] ===
Integrated Visualization Framework: Connects all audit aspects — risk assessment, controls, substantive testing, and reporting — to specific audit objectives through an interactive tree map (AuditTreeD3) and COSO radar chart.

=== PARA 24 [STYLE:List Paragraph] ===
Pre-Request Visibility: AI-powered SampleLens indicates sample size ranges (x-y) BEFORE selection, with document type predictions (POs, invoices, contracts) and methodology explanations.

=== PARA 25 [STYLE:List Paragraph] ===
Comprehensive KPI Integration: Dashboard aggregates KPIs, risk heatmaps, status distributions, and activity timelines, giving clients an upfront view of anticipated requests across all audit objectives.

=== PARA 26 [STYLE:List Paragraph] ===
Dual Progress Visualization: Tracks both auditor progress through the audit lifecycle AND client PBC submission progress, with overdue alerts and real-time status dashboards.

=== PARA 27 [STYLE:List Paragraph] ===
Impact Analysis Engine: ImpactSimulator uses Internal Audit Methodology graph BFS propagation to quantify the ripple effect of changes or delays, with AI-recommended next steps and mitigation strategies.

=== PARA 28 [STYLE:List Paragraph] ===
Role-Based Visibility Control: VisibilityPanel enables engagement teams to manage what is visible to each stakeholder (Partner, CFO, Auditor, Finance Clerk) with fine-grained data filtering.

2.2 Four AI Pipelines

=== PARA 43 [STYLE:Normal] ===
Each AI pipeline follows a consistent architecture: YAML template loading → LLM invocation (eg.DeepSeek) → structured JSON parsing → Pydantic validation. All pipelines feature automatic Mock fallback for zero-configuration demonstrations.

=== PARA 44 [STYLE:List Paragraph] ===
Risk Analyzer — COSO-based risk assessment with residual risk scoring across five domains. Currently Currently uses DeepSeek LLM (temperature 0.3) for stable, consistent output.

=== PARA 45 [STYLE:List Paragraph] ===
Sample Recommender — Statistical sampling engine incorporating control frequency, operating type multipliers, historical deficiencies, and RAWTC risk scoring. Conditional branch logic: low-risk scenarios skip LLM for simplified results. Currently Currently uses DeepSeek LLM (temperature 0.2) for precision.

=== PARA 46 [STYLE:List Paragraph] ===
Impact Simulator — BFS propagation on Internal Audit Methodology graph (propagation factor 0.6) with mitigation strategy comparison (mitigate/accept/transfer/avoid). Currently uses DeepSeek LLM (temperature 0.3) for structured graph data.

=== PARA 47 [STYLE:List Paragraph] ===
Narrative Generator — Audience-adapted communication engine transforming technical findings into differentiated narratives. Currently uses DeepSeek LLM (temperature 0.5) balancing creativity with controllability.

4. Technical Architecture, Tools & Prompt Engineering

=== PARA 74 [STYLE:Heading 2] ===
4.1 System Architecture

=== PARA 75 [STYLE:Normal] ===
LuminFlow employs a dual-stack architecture with frontend-backend separation communicating via HTTP RESTful APIs:

=== PARA 76 [STYLE:List Paragraph] ===
Frontend: React 18 + TypeScript + Vite (port 5173). Component-Store-Service MVC pattern with Zustand state management, TanStack React Query for data caching, and role-based route guards.

=== PARA 77 [STYLE:List Paragraph] ===
Backend: FastAPI + Python 3.x (port 8000). Route-Engine-Model layered architecture with SSE streaming for real-time AI chat, async HTTP clients, and graceful Mock degradation.

=== PARA 78 [STYLE:List Paragraph] ===
Visualization: D3.js for force-directed graphs and tree maps, ECharts for radar/heatmap/sunburst charts, Recharts for trend analysis.

=== PARA 79 [STYLE:List Paragraph] ===
Internationalization: Custom i18n system (Zustand + localStorage) with zero external dependencies, supporting Chinese/English bilingual UI and AI-generated content.

4.3 Prompt & Context Engineering

=== PARA 83 [STYLE:Normal] ===
LuminFlow's AI pipelines are driven by a modular prompt engineering system:

=== PARA 84 [STYLE:List Paragraph] ===
YAML Template Architecture: Each AI pipeline has a dedicated YAML template with separation of system_prompt (role definition), user_template (dynamic data injection), and output_format (structured JSON schema). Templates are version-controlled and independent.

=== PARA 85 [STYLE:List Paragraph] ===
Context Injection: Chat requests automatically append the user's role, current page context, and selected data elements, enabling the AI to provide contextually relevant responses without manual prompt engineering.

=== PARA 86 [STYLE:List Paragraph] ===
Audience Adaptation: Narrative prompts instruct the LLM to rephrase technical audit findings for three personas: CFO (financial impact, business risk), Internal Audit (control effectiveness, remediation steps), IT (system dependencies, configuration changes).

=== PARA 87 [STYLE:List Paragraph] ===
Temperature Tuning: 0.2 for sampling (precision-critical calculations), 0.3 for risk analysis and impact simulation (stability), 0.5 for narrative generation (creativity within structure).

=== PARA 88 [STYLE:List Paragraph] ===
Structured Output Enforcement: All LLM responses are required in JSON format matching Pydantic models. Failed parsing triggers automatic fallback to realistic mock data, ensuring the UI never breaks.

=== PARA 89 [STYLE:List Paragraph] ===
Graceful Degradation: Every AI pipeline includes a comprehensive mock fallback with professionally curated sample data. This enables complete platform demonstrations without any API key configuration.
